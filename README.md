# ThreeXThreePickaxe

Плагин для Paper/Spigot **1.21.8**, добавляющий кастомные кирки,
которые ломают блоки выбранной областью (3x3, 5x5, 9x9 или другой
размер, настроенный в конфиге). Область строится в плоскости,
перпендикулярной направлению взгляда игрока (копаете вниз — область
горизонтальная, копаете в стену — вертикальная и т.д.).

## Команды

```
/3x3 <wooden|stone|iron|golden|diamond|netherite> <радиус> [игрок]
```

Радиус — любое значение из списка `allowed-radii` в config.yml
(по умолчанию 3, 5, 9). Без указания игрока кирка выдаётся себе.

```
/3x3reload
```

Перечитывает `config.yml` без перезапуска сервера.

Примеры:

```
/3x3 diamond 9            — себе алмазная кирка 9x9
/3x3 iron 5 Steve         — игроку Steve железная кирка 5x5
```

## Права

| Право                      | За что отвечает                              | По умолчанию |
|-----------------------------|-----------------------------------------------|--------------|
| `threexthree.give`          | Получать кирку себе                          | op           |
| `threexthree.give.others`   | Выдавать кирку другим игрокам                | op           |
| `threexthree.reload`        | Перечитывать config.yml                      | op           |

## Конфиг (`plugins/ThreeXThreePickaxe/config.yml`)

Создаётся автоматически при первом запуске сервера с этим плагином.

```yaml
blacklisted-blocks:   # блоки, которые НЕ ломаются "за компанию"
  - CHEST
  - SPAWNER
  - ...

allowed-radii:        # какие радиусы можно указывать в команде /3x3
  - 3
  - 5
  - 9

break-sound:           # звук при поломке кирки от износа
  enabled: true
  sound: ENTITY_ITEM_BREAK
  volume: 1.0
  pitch: 1.0
```

После изменения файла выполните `/3x3reload` — перезапуск сервера
не требуется.

## Прочность

Прочность кирки расходуется за каждый дополнительно сломанный блок
(с учётом шанса от Unbreaking). Чем больше радиус — тем больше
блоков ломается за один удар, и тем быстрее садится прочность.

## Возможности

- Учитывается прочность инструмента и зачарование Unbreaking.
- Соблюдаются дроп предметов (с учётом Fortune/Silk Touch — это
  делает стандартный `block.breakNaturally(tool)`).
- Дополнительные блоки ломаются через "фейковое" событие
  `BlockBreakEvent`, поэтому защита территорий (WorldGuard и
  аналоги) и другие плагины, слушающие это событие, продолжают
  работать как обычно. При этом есть защита от рекурсии: листенер
  не реагирует на свои же искусственные события.
- Блок-лист контейнеров и прочих "важных" блоков настраивается
  через config.yml.

## Сборка

Нужен JDK 21 и Maven.

```bash
mvn clean package
```

Готовый jar появится в `target/threexthree-pickaxe.jar`. Либо
используйте GitHub Actions (workflow `.github/workflows/build.yml`
уже настроен — просто запустите его на вкладке Actions).
